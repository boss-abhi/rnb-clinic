## 2. Architecture

```
┌───────────────────────────────────────────────────────┐
│                    PUBLIC INTERNET                      │
└────────────────────────┬──────────────────────────────┘
                         │ HTTPS :443
          ┌──────────────▼──────────────┐
          │   Apache/Nginx (Plesk)       │
          │   SSL via Let's Encrypt      │
          │   thernbclinic.com → :3000   │
          │   api.thernbclinic.com       │
          │    or /strapi → :1337        │
          └──────────────┬──────────────┘
                         │
          ┌──────────────▼──────────────┐
          │   Next.js 15 (Node.js)       │
          │   Port 3000                  │
          │   App Router, SSG + ISR      │
          │   revalidate: 300s           │
          └──────────────┬──────────────┘
                         │ REST (internal)
          ┌──────────────▼──────────────┐
          │   Strapi CE (Node.js)        │
          │   Port 1337                  │
          │   Admin: /admin (no public   │
          │   link, no registration)     │
          └──────────────┬──────────────┘
                         │
          ┌──────────────▼──────────────┐
          │   MySQL 8.x (Plesk MySQL)    │
          │   DB: rnbclinic_strapi       │
          └─────────────────────────────┘

Side channels:
  - Lead/Appointment → Strapi DB + email via nodemailer/SMTP
  - Uploads → Strapi /uploads (local provider or S3-compatible)
  - Maps → Google Maps iframe (URL stored in Site Settings CMS)
```

**Key decisions:**

| Concern | Decision |
|---|---|
| Rendering | SSG for all content pages, ISR `revalidate: 300` for dynamic content, SSR not needed |
| API auth | Strapi API token (read-only, stored in Next.js `.env.local`) |
| Form submission | Next.js API route → Strapi REST → email via SMTP |
| Images | `next/image` with Strapi hostname whitelisted |
| State | No global state needed; server components by default |
| Rate limiting | `@strapi/plugin-rate-limiter` + Next.js API route limit via `rate-limiter-flexible` |
| Spam | Honeypot field + timestamp delta check (no reCAPTCHA dependency) |

## 3. Phase Plan

### Phase 0 — Project Bootstrap (Day 1)
- [ ] Init monorepo structure (`frontend/`, `strapi/`, `docs/`)
- [ ] Create Next.js 15 app: `npx create-next-app@latest frontend --typescript --tailwind --app`

## 6. Design System

### Color Tokens (`tailwind.config.ts`)
```ts
colors: {
  brand: {
    blue:    { DEFAULT: '#1A6FA8', light: '#3B8FCA', dark: '#0F4F7A' },
    green:   { DEFAULT: '#2EAD7A', light: '#4CC895', dark: '#1D8A5E' },
    neutral: { 50: '#F8FAFB', 100: '#EEF2F5', 900: '#1A2332' },
  }
}
```

### Typography
- Headings: `Inter` (or `Plus Jakarta Sans`) — clean, medical-premium
- Body: `Inter`
- Loaded via `next/font/google`, subset Latin

### Component Patterns
- Cards: `rounded-2xl shadow-sm hover:shadow-md transition-shadow bg-white border border-neutral-100`
- Buttons: primary (brand-blue), ghost, outline variants
- Sections: alternating `bg-white` / `bg-brand-neutral-50` for visual rhythm
- Gradients: `from-brand-blue/5 to-brand-green/5` for hero overlays
- Transitions: `duration-200 ease-out` — subtle, no janky animations
